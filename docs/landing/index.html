Per-org guardrails

Production stays read-only until you decide otherwise.

Every org has an explicit safety level. Actions outside that level are hidden or blocked, and the active level stays visible while you work.

01Read-onlyBrowse, query, compare, and export
02RecordsAllow record changes
03MetadataAllow metadata writes and deploys
04FullAllow destructive operations and anonymous Apex

Salesforce still enforces the connected user's permissions. sf-deck adds a local guardrail; it does not replace platform security.

The same guardrails, scriptable

Automate the workflow without inventing another API.

Core operations expose versioned JSON, predictable exit codes, and the same per-org safety checks. Scripts and agents can also control a running sf-deck window through its local Unix socket.
