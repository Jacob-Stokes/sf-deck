Salesforce, without the tab maze

Every Salesforce org,
one keystroke away.

Explore schema, records, code, users, deploys, and org health from one keyboard-first terminal workspace. sf-deck uses your existing Salesforce CLI sessions and keeps its working state local.

  • Uses existing sf sessions
  • No package or connected app
  • No application telemetry

Built for the work between orgs

Four workflows, recorded against fictional Northwind orgs.

01
Multi-org by design

Switch orgs. Each keeps its place.

Move from development to UAT to production without rebuilding your workspace. Each org remembers its own context.

Follow the cross-org workflow →
three fictional Northwind orgsDEMO
02
One object, the whole picture

Stop opening six Setup pages.

Drill into an object once, then move through its fields, records, validation rules, record types, triggers, and field-level security. The object stays anchored while the question changes.

Explore objects and records →

03
Search across the workspace

Find it before you remember where Salesforce put it.

Press Ctrl+F to search objects, fields, flows, Apex, users, and other loaded metadata from one place. Open the result directly and keep working.

Learn the keyboard workflow →

04
From investigation to working set

Turn discoveries into a project.

Collect objects, fields, flows, Apex, records, and saved queries into a dev project as you investigate. Review the working set, tag it, and create an SFDX bundle when it is ready to move.
